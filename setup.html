導入ガイド

server-bot の設置から初回起動、Discord bot として動き始めるまでの手順です。

1. 設置

server.py を任意の場所に配置します。

推奨ディレクトリは、実行する server.[exe/jar] が存在する階層です。 その場所が root ディレクトリとなり、一部の権限を持つ Discord ユーザーはファイル操作 (/cmd stdin 系コマンド)を行えます。
ただし server.exe や server.jar 本体が存在する階層は、 ドライブ直上(Windows の場合 C:\ 直下など)でない必要があります。 必ず何らかのディレクトリの中に入れてください。 これは初期状態では ../backup/ 内にバックアップが生成されるためです。

2. 実行

依存ライブラリをインストールしてから server.py を実行します。

pip install -r requirements.txt
python server.py

実行すると、server.py と同じ階層に .config と .token が生成されます。

コンソール引数(任意)

エラーが発生する場合などに利用してください。

引数効果
-initupdate.py など、server.py からダウンロードされる一部ファイルを再度ダウンロードします。
.token が生成されない場合、.config 内の server_path + server_name が存在していません。サーバーが存在するパスと、拡張子を含むサーバーのファイル名に修正してください。

3. 初期設定

  1. .token に Discord bot のトークンを記述する
  2. .config の server_path に server.[exe/bat](jar を実行する場合は bat)へのパスを記述する
  3. 再度 server.py を起動する

このとき同時に mikanassets ディレクトリが生成されます。 これは自己アップデートや Web コンソールへのアクセスに必要なファイル群です。

このプログラムは server.py がサーバー本体を呼び出すため、 server.[exe/jar/bat] を自身で起動する必要はありません。 そのほかの設定項目は .config リファレンス を参照してください。

4. Java 版サーバーの起動について

Java 版サーバーを Windows で起動する場合、以下の内容の bat ファイルを作成し、 .config の server_name に設定してください (fabric なら start.bat、forge なら run.bat が一般的です)。 

java -Xmx4048M -Xms1024M -Dfile.encoding=UTF-8 -jar server.jar nogui
  • nogui オプションが無い場合、現在 /stop 等を利用できません。
  • -Dfile.encoding=UTF-8 が無い場合、一部環境で特殊文字等が正常に表示されません。
  • pause のようなコマンドを bat に記載しないでください。このプログラムはサーバーを制御するためのものであり、サーバー以外のコマンドを実行しないでください。

5. アップデート

.config の update.auto が true の場合、起動時に GitHub の最新コミットを確認して自動更新します。 Discord から /update を実行して手動で更新することもできます。 参照するブランチは update.branch で変更できます(通常は main のままにしてください)。
