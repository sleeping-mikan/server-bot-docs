導入ガイド

server-bot の設置から初回起動、Discord bot として動き始めるまでの手順です。 プログラミングの経験がなくても進められるように、必要な準備から順に説明します。 途中でエラーが出た場合は、ページ末尾のつまずきやすいポイントと用語集も参照してください。

0. 前提条件 — 最初にそろえるもの

server-bot を動かすには、次の 4 つが必要です。すでに用意できているものは読み飛ばしてください。

(1) Python 3.12

server-bot は Python というプログラミング言語で書かれたプログラムです。 動かすにはお使いのパソコンに Python 本体(バージョン 3.12.x)をインストールする必要があります。 Python 公式サイトからインストーラーをダウンロードして実行してください。

Windows のインストーラーでは、最初の画面で「Add python.exe to PATH」に必ずチェックを入れてください。 チェックを入れ忘れると、後の手順で python コマンドがコマンドプロンプトから認識されません。

(2) Discord bot(アプリケーション)とトークン

bot を動かすには「トークン」と呼ばれる長い英数字の文字列が必要です。 これは bot 本人であることを証明する、パスワードのようなものです。 トークンは Discord Developer Portal で bot を作成すると発行されます。 大まかな流れは次のとおりです(画面の詳細な操作は Discord 公式ドキュメント(英語)を参照してください)。

  1. Discord Developer Portal にログインし、「New Application」から新しいアプリケーション(bot)を作成する
  2. 左メニューの「Bot」ページでトークンを発行(Reset Token)し、コピーして控えておく
  3. 同じ「Bot」ページの Privileged Gateway Intents にある「Message Content Intent」を必ずオンにする
トークンは絶対に他人に教えたり、インターネット上に公開したりしないでください。 トークンを知っている人は誰でもあなたの bot を操作できます。 誤って公開してしまった場合は、Developer Portal で Reset Token を行い、新しいトークンに差し替えてください。

(3) bot を自分の Discord サーバーに招待する

作成した bot は、招待 URL から自分の Discord サーバーに追加しておきます。 Developer Portal の「OAuth2」ページにある URL Generator で bot と applications.commands にチェックを入れて URL を生成し、 その URL をブラウザで開いて追加先のサーバーを選択してください。

(4) 管理したいゲームサーバー本体

server-bot はゲームサーバーを「操作する」ツールであり、ゲームサーバーそのものは含まれていません。 Minecraft 統合版なら bedrock_server.exe、Java 版なら server.jar など、 動かしたいサーバー本体を各公式サイトから入手してください。 先に一度サーバー単体で起動できることを確認しておくと、後の切り分けが楽になります。

1. server.py の入手と設置

GitHub リポジトリから server.py と requirements.txt の 2 ファイルを入手します (緑色の「Code」ボタン →「Download ZIP」でリポジトリ全体をダウンロードし、その中からこの 2 ファイルを取り出すのが簡単です)。

入手した server.py と requirements.txt を、 ゲームサーバー本体(server.exe / server.jar など)と同じフォルダにコピーします。

推奨の設置場所は、実行する server.[exe/jar] などが存在するフォルダ(階層)です。 その場所が bot にとっての root ディレクトリ(ファイル操作の起点)となり、 一部の権限を持つ Discord ユーザーはこのフォルダ以下に対してファイル操作 (/cmd stdin 系コマンド)を行えます。
ただし server.exe や server.jar 本体が存在するフォルダは、 ドライブ直上(Windows の場合 C:\ 直下など)でない必要があります。 必ず何らかのフォルダの中に入れてください。 これは初期状態では 1 つ上のフォルダの backup/ 内にバックアップが生成されるためです。

2. 実行

コマンドプロンプト(ターミナル)を開き、server.py を置いたフォルダで次の 2 つのコマンドを順に実行します。 1 行目は依存ライブラリ(bot の動作に必要な追加部品)のインストールで、初回のみ必要です。 2 行目で server-bot が起動します。

Windows でのコマンドプロンプトの開き方: エクスプローラーで server.py のあるフォルダを開き、 アドレスバー(上部のパス表示欄)に cmd と入力して Enter を押すと、 そのフォルダを開いた状態でコマンドプロンプトが起動します。 
pip install -r requirements.txt
python server.py

初回実行では、server.py と同じフォルダに設定ファイル .config と .token が生成され、 「please write token in ...」に続いて「please press any key to continue...」と表示されて停止します。 これは正常な動作です。 何かキーを押すとプログラムが終了するので、続けて「3. 初期設定」に進んでください。

このとき「not exist ○○ file. please check your config.」というエラー (○○ は bedrock_server.exe のパス)が同時に表示されることがあります。 これは server_name の初期値が bedrock_server.exe(Minecraft 統合版)のためで、 別のサーバーを使う場合はこの時点で表示されるのが正常です。 「3. 初期設定」で server_name を書き換えると表示されなくなります。
プログラム本体は mikanassets/main/ に配置されます。 初回実行時など本体が存在しない場合は、GitHub から自動でダウンロードされます。 本体のファイルが壊れた場合は mikanassets/main/ を削除して server.py を再実行すると、再ダウンロードされます。
.token が生成されず「not exist server_path dir」と表示される場合は、 .config 内の server_path のフォルダが存在していません。 サーバー本体が存在するフォルダのパスに修正して再実行してください。

3. 初期設定

  1. .token をメモ帳などのテキストエディタで開き、「ここにtokenを入力」と書かれた内容を消して、前提条件 (2) で控えた Discord bot のトークンに書き換えて保存する
  2. .config をメモ帳などで開き、server_name をサーバー本体のファイル名(拡張子まで含む。例: bedrock_server.exe。Java 版の jar を bat 経由で起動する場合はその start.bat など)に書き換える
  3. server.py をサーバー本体と別のフォルダに置いた場合のみ、server_path をサーバー本体があるフォルダのパスに書き換える
  4. 再度 python server.py を実行する
server.py をサーバー本体と同じフォルダに置いた場合、server_path は 自動的にそのフォルダに設定されているため、変更は不要です。
.config は JSON というテキスト書式で書かれています。値は "..."(半角ダブルクォート)で囲み、 Windows のパスに含まれる \ は \\ と 2 つ重ねるか、代わりに / を使ってください (例: "D:\\a\\" または "D:/a/")。

初回起動時には mikanassets ディレクトリも生成されます。 これは自己アップデートや Web コンソールへのアクセスに必要なファイル群です。

このプログラムは server.py がサーバー本体を呼び出すため、 server.[exe/jar/bat] を自身で起動する必要はありません。 そのほかの設定項目は .config リファレンス を参照してください。

4. 動作確認

起動に成功すると、Discord 上で bot がオンライン表示になり、ゲームサーバー本体も自動的に起動します。 bot を招待した Discord サーバーのテキストチャンネルで、メッセージ入力欄に / を入力すると /status や /help などのコマンド候補が表示されます。 まずは /status を実行して、サーバーの状態が表示されるか確認してみてください。 使えるコマンドは コマンド一覧 を参照してください。

コマンド候補が表示されない場合は、招待 URL に applications.commands が含まれていたかを確認し、 数分待ってから Discord を再起動してみてください(コマンドの反映には時間がかかることがあります)。

5. アップデート

.config の update.auto が true の場合、起動時に GitHub の最新コミットを確認して自動更新します。 Discord から /update を実行して手動で更新することもできます。 参照するブランチ(GitHub 上のバージョン系統の名前)は update.branch で変更できます(通常は main のままにしてください)。

補足: Java 版 Minecraft サーバー(server.jar)を起動する場合

この bot は Minecraft 専用ツールではありませんが、Java 版サーバーを Windows で動かす場合は 以下の内容の bat ファイル(起動用の命令を書いたテキストファイル)を作成し、 .config の server_name にそのファイル名を設定してください (fabric なら start.bat、forge なら run.bat が一般的です)。 bat ファイルは、メモ帳に以下の 1 行を書き、server.jar と同じフォルダに start.bat のような「.bat」で終わる名前で保存すると作成できます。 

java -Xmx4048M -Xms1024M -Dfile.encoding=UTF-8 -jar server.jar nogui

nogui が無いと /stop 等を利用できません。 -Dfile.encoding=UTF-8 が無いと一部環境で特殊文字等が正常に表示されません。 また pause のようなコマンドを bat に記載しないでください (このプログラムはサーバーを制御するためのものであり、サーバー以外のコマンドを実行しないでください)。

つまずきやすいポイント

起動時によくあるエラーメッセージと対処方法です。

起動時のエラーで停止した場合は「please press any key to continue...」と表示され、キー入力待ちになります。 何かキーを押すとプログラムが終了するので、下表の対処を行ってから再実行してください (ウィンドウを閉じて終了しても問題ありません)。
症状 / 表示されるメッセージ原因と対処
'python' は、内部コマンド…として認識されていません Python がインストールされていないか、PATH に登録されていません。前提条件 (1) に戻り、インストーラーで「Add python.exe to PATH」にチェックを入れて再インストールしてください。
import error. please run 'pip install -r requirements.txt' 依存ライブラリが不足しています。server.py のあるフォルダで pip install -r requirements.txt を実行してください。
not exist server_path dir .config の server_path に書かれたフォルダが存在しません。サーバー本体があるフォルダのパスに修正してください。
not exist ○○ file. please check your config server_path のフォルダ内に server_name のファイルが見つかりません。server_name が実際のファイル名(拡張子まで含む)と一致しているか確認してください。
please write token in … / token が未設定です .token にトークンが記入されていません。初期設定の手順 1 を行ってから再実行してください。
トークンが無効です .token の内容が正しくありません。前後に余計な文字や空白が入っていないか確認するか、Discord Developer Portal でトークンを再発行(Reset Token)して貼り直してください。
Discord で / を打ってもコマンド候補が出ない bot の招待 URL に applications.commands が含まれていなかった可能性があります。前提条件 (3) の URL を作り直して再招待し、数分待ってから Discord を再起動してください。

用語集

このドキュメントに登場する用語の簡単な説明です。

用語意味
コマンドプロンプト / ターミナル文字を打ち込んでパソコンに命令するための画面。Windows では「cmd」とも呼ばれます。
pipPython に付属する、ライブラリ(追加部品)をインストールするためのコマンド。Python を入れると一緒に使えるようになります。
トークンbot 本人であることを証明する長い英数字の文字列。パスワードと同じように秘密にして扱います。
パスファイルやフォルダの場所を表す文字列。例: C:/servers/minecraft/
ディレクトリ / 階層「フォルダ」のこと。「同じ階層」は「同じフォルダの中」という意味です。
root ディレクトリbot がファイル操作を行うときの起点となるフォルダ。サーバー本体(と server.py)を置いたフォルダを指します。
JSON設定などを書くためのテキストの書式。.config で使われています。
標準入力起動中のサーバーのコンソール(黒い画面)に直接コマンドを打ち込むのと同じ入力のこと。
ポート番号 / ポート開放ポート番号は通信の窓口の番号。ポート開放は、自宅のルーターを設定して外部からその窓口への通信を許可することです。
ブランチGitHub 上でのバージョンの系統名。通常は main(安定した最新版)を使います。
リバースプロキシ外部からのアクセスを代わりに受けて転送してくれるサーバー。https 化などに使います(上級者向け)。
