コマンド一覧

Discord bot として利用できる全コマンドです。 Discord のメッセージ入力欄に / を入力すると候補が表示されるので、選択して実行できます。 必要権限レベルは .config の commands_level から変更できます。

表の見方: <path> のような山かっこ付きの表記は、実行時に指定する引数(入力値)です。 ? が付いた引数は省略できます。 「初期権限」列は実行に必要な権限レベルの初期値で、数字が大きいほど強い権限が必要です (レベル 0 のコマンドは誰でも実行できます)。 権限レベルの仕組みはホームの「権限レベルについて」を参照してください。
/

サーバー操作

コマンド実行結果初期権限
/start サーバーを開始します。なお server.py 起動時には自動的に開始されます。 1
/stop サーバーを停止します。server.py は実行状態のままなので、他のコマンドは引き続き使用できます。 1
/restart bot(server.py)を再起動します。アップデートは行いません。サーバー停止中にのみ使用できます。 2
/exit server.py を終了します。次回サーバー管理者が server.py を起動するまで bot を利用できなくなります。サーバー停止中にのみ使用できます。 2
/cmd serverin <server command> サーバーに対してコマンドを送信します(.config の allow_cmd で許可されたもののみ)。 1

バックアップとファイル操作

コマンド実行結果初期権限
/backup create <path> サーバーデータをバックアップします。バックアップする対象(フォルダやファイル)を root ディレクトリからの相対パスで指定します(例: world)。 1
/backup apply <which> <path?> バックアップを指定したディレクトリに展開します。path が与えられない場合は root ディレクトリ(サーバー本体のフォルダ)に展開します。 3
/cmd stdin ls <path> ディレクトリ内のファイル一覧を返します。root ディレクトリはサーバー直上になります。 2
/cmd stdin mk <path> <file?> ファイルを作成または上書きします。file 引数を渡さない場合は空のファイルを作成します。 3*1
/cmd stdin rm <path> 指定されたファイルが存在した場合削除します。 2*1
/cmd stdin rmdir <path> 指定されたディレクトリが存在した場合削除します。ディレクトリ内に対して再帰的に適用されます。 2*1
/cmd stdin mkdir <path> 指定されたディレクトリを作成します。 2*2
/cmd stdin send-discord <path> 指定されたファイルを Discord に送信します。 2*2
/cmd stdin wget <url> <path?> 指定のパスのファイルを url から得られたデータで上書きします。path 引数を渡さない場合は直上にファイルを生成します。 3*1
/cmd stdin mv <src> <dest> 指定した path を dest に移動します。 3*1
/cmd stdin unzip <path> 指定した .zip ファイルをそのディレクトリに展開します。 2*2

情報と権限

コマンド実行結果初期権限
/status 現在のサーバーの状態を表示します。 0
/help Discord 上に help と web リンクを表示します。web サイトへ外部からアクセスするには追加のポート開放が必要です。 0
/ip この bot(サーバー)が実行している IP アドレスを返します。 0config で有効/無効を切替
/logs <file?> サーバーログを表示します。引数が与えられた場合は該当ファイルから、無い場合は現在のサーバーログを表示できる限り表示します。 1
/permission view <user> <detail?> プレイヤーの権限を表示します。 0
/permission change <level> <user> ユーザーの bot 利用権限を操作します。 4

運用・その他

コマンド実行結果初期権限
/terminal set <ch?> Discord のチャンネル ID を与えると、サーバーのログをそのチャンネルに送信します。また標準入力を受け取ります。 1
/terminal del ターミナルの紐づけを解除します。 1
/tokengen web アクセスのためのトークンを生成します。web ログイン画面で入力してください。 1
/update <is_force?> GitHub にアクセスして最新の本体をダウンロードします。is_force を True にすると強制的にダウンロードします。 3
/lang <"ja"/"en"> サーバーの言語を変更します。 2
/announce embed <file/txt> 通常のテキストまたは mimd 形式のテキスト/ファイルを指定して、bot として Discord にメッセージを送信します。テキストの場合、改行には \n を利用してください。 4

注記

*1 : 一部のファイル/ディレクトリに対する操作は行いません。 ただし Discord 管理者権限を保持した上で、.config の enable_advanced_features を true にすることで操作可能となります。

*2 : 一部のファイル/ディレクトリに対する操作は行いません。

/terminal set で指定するチャンネル ID は、Discord の設定で「開発者モード」を有効にすると、 チャンネル名の右クリックメニュー(「ID をコピー」)から取得できます。

各コマンドに関係する設定は .config リファレンス を参照してください。 「標準入力」「パス」などの用語は導入ガイドの用語集で説明しています。
