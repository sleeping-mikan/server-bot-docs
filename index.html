server-bot ドキュメント

server-bot は、Minecraft(統合版 / Java 版)などのゲームサーバーを Discord から起動・停止・操作・バックアップできる bot です。 サーバーログの Discord への転送、Web ブラウザからの操作、拡張機能による機能追加にも対応しています。

クイックスタート

あらかじめ Python 3.12 のインストールと Discord bot(トークン)の作成が必要です。 これらの準備がまだの方は、導入ガイドの「前提条件」から読み始めてください。

  1. GitHub から入手した server.py と requirements.txt を、サーバー本体(server.exe / server.jar など)と同じフォルダに配置する
  2. コマンドプロンプト(ターミナル)でそのフォルダを開き、pip install -r requirements.txt で依存ライブラリをインストールする
  3. python server.py を一度実行し(キー入力待ちで停止するので、何かキーを押して終了する)、生成された .token に Discord bot のトークンを記述する
  4. .config の server_path / server_name を確認・修正する
  5. 再度 python server.py を実行する — サーバー本体は bot が自動で起動します

各手順の詳しいやり方と注意点は 導入ガイド を参照してください。

権限レベルについて

権限レベルとは、bot がユーザーごとに持つ「どこまで操作を許すか」を表す数値で、 数字が大きいほど強い操作(bot の終了やファイルの削除など)ができます。 各コマンドには必要権限レベルが設定されており、 実行にはそのレベル以上の権限が必要です(レベル 0 のコマンドは誰でも実行できます)。 ユーザーの権限は /permission change で変更でき、必要レベル自体も .config の commands_level で調整できます。 初期状態では Discord サーバーの管理者権限を持つユーザーが最高権限として扱われます。
