拡張機能

server-bot は拡張機能によって独自の Discord コマンドや定期タスクを追加できます。 このページでは v3 における拡張機能の実装方法を説明します。

このページは Python で bot の機能を自作したい開発者向けの内容です。 bot を普通に使うだけであれば読む必要はありません。 導入がまだの方は導入ガイドからどうぞ。

配置場所

server.py と同じディレクトリの mikanassets/extension/ 以下に、 拡張機能ごとのディレクトリを作って配置します。 ディレクトリ直下の commands.py がエントリポイントとして読み込まれます。 

mikanassets/
└── extension/
    └── 拡張機能名/
        ├── commands.py   ← エントリポイント(必須)
        └── その他のファイル(任意)

拡張機能は server.py の起動時に読み込まれます。 追加・変更した場合は server.py を再起動してください。 commands.py が存在しないディレクトリや、読み込み中に例外が発生した拡張機能は スキップされ、ログにその旨が出力されます(bot 本体は起動を継続します)。

拡張機能は bot 本体と同じ権限で動く Python コードです。 配布されている拡張機能を導入する場合は、内容を理解した上で自己責任で使用してください。

コマンドを追加する

bot は拡張機能を読み込む直前に、その拡張機能専用のコマンドグループ /extension-拡張機能名 を作成し、ctx.extension_commands_group にセットします。 commands.py ではこれを受け取ってコマンドを登録します。

最小構成の例(mikanassets/extension/template/commands.py):

import discord

from core.state import ctx

# ロード時にセットされる値は、必ず import 時点(モジュール先頭)で変数に保持する
tree   = ctx.extension_commands_group
logger = ctx.extension_logger

# /extension-template hello として登録される
@tree.command(name="hello", description="Say hello")
async def hello_command(interaction: discord.Interaction):
    await interaction.response.send_message("Hello, world!")
    logger.info("hello sent")
ctx.extension_commands_group と ctx.extension_logger は 「いま読み込んでいる拡張機能」のために一時的にセットされる値で、全拡張機能のロード完了後にはクリアされます。 関数の中から ctx.extension_commands_group を直接参照せず、 上の例のようにモジュール先頭で変数に受けてください。

コマンドグループの名前は拡張機能のディレクトリ名から自動で決まります。 ディレクトリ名が template なら、コマンドは Discord 上で /extension-template hello のように表示されます。

サブコマンドグループ

discord.app_commands.Group を作って tree に登録すると、さらに階層を作れます。

# /extension-template group say <message> として登録される
my_group = discord.app_commands.Group(name="group", description="A group of commands")

@my_group.command(name="say", description="Say something")
async def say_command(interaction: discord.Interaction, message: str):
    await interaction.response.send_message(message)

tree.add_command(my_group)

権限チェックとユーザーログ

bot 本体と同じ権限システム・ログ形式を利用できます。

from bot.utils import print_user, user_permission, not_enough_permission

@tree.command(name="secret", description="Level 2 or higher only")
async def secret_command(interaction: discord.Interaction):
    await print_user(logger, interaction.user)      # 利用者をログに残す
    if await user_permission(interaction.user) < 2: # 権限レベルを確認
        await not_enough_permission(interaction, logger)
        return
    await interaction.response.send_message("ok")

定期タスクを追加する

discord.ext.tasks のループを append_task() で登録すると、 bot の on_ready 後に自動で start() されます。 

from discord.ext import tasks

from bot.extensions import append_task

@tasks.loop(seconds=60)
async def heartbeat():
    logger.info("alive")

append_task(heartbeat)

API リファレンス

拡張機能から利用できる関数の詳細です。 bot 本体のモジュールは技術的にはどれでも import できてしまいますが、 ここに載せている API は「外部から呼んでも bot 内部の状態を壊さない」ように作られている入口です。 特にサーバープロセスへの操作は、必ずこれらの API を経由してください (たとえば Popen の標準入力へ直接書き込むと、bot 本体のコマンド処理と書き込みが 割り込み合い、サーバーに壊れたコマンドが届くことがあります)。

bot.extensions.write_server_in(command)

write_server_in(command: str) -> tuple[bool, str]

管理対象サーバーの標準入力(コンソール)に command を 1 行のコマンドとして書き込みます。 改行は自動で付加されるため、command に含める必要はありません。 内部の書き込みロックを取得してから書き込むため、bot 本体の /cmd serverin や 他の拡張機能からの書き込みと競合しません。

戻り値意味
(True, "success")書き込みに成功しました。
(False, "server_is_not_running")サーバーが起動していません。
(False, "write_server_block")他の書き込み処理が実行中です。少し待ってから再試行してください。
戻り値の bool を必ず確認してください。サーバー停止中に送ったコマンドは失敗し、送信されません。

bot.extensions.append_task(func)

append_task(func: discord.ext.tasks.Loop) -> None

@tasks.loop(...) で作った定期実行ループを bot に登録します。 登録されたループは bot の on_ready(Discord への接続完了)後に自動で start() され、 /exit や /restart、アップデートによる終了時には自動で cancel() されます。 起動・停止のタイミングを bot 側が管理するため、自分で start() / cancel() を呼ぶ必要はありません。

登録は commands.py の読み込み時(モジュール先頭)に行ってください。 bot の起動完了後に登録してもループは開始されません。

bot.utils.print_user(logger, user)

async print_user(logger: logging.Logger, user: discord.User) -> None

「誰がコマンドを実行したか」を command used by ユーザー名 という形式で logger に INFO として記録します。bot 本体の全コマンドがこの形式でログを残しているため、 拡張機能のコマンドでも先頭で呼んでおくと、ログの形式がそろい追跡しやすくなります。 await が必要です。

bot.utils.user_permission(user)

async user_permission(user: discord.User) -> int

ユーザーの権限レベルを整数で返します。await が必要です。判定の仕組みは bot 本体のコマンドと同じです:

  • .config の admin.use_discord_admin が true(初期値)の場合、 Discord サーバーの管理者権限を持つユーザーには最大の権限レベルが返ります
  • それ以外は /permission change で設定された権限テーブルの値が返ります(未登録ユーザーは 0)

「レベル 2 以上のみ実行可」のようにしたい場合は、 権限チェックとユーザーログのコード例のように比較してください。

bot.utils.not_enough_permission(interaction, logger)

async not_enough_permission(interaction: discord.Interaction, logger: logging.Logger) -> None

権限不足時の定型応答を返します。bot 本体と同じ「権限がありません」のエラー embed を、 実行した本人にだけ見えるメッセージ(ephemeral)として送信し、logger にエラーを記録します。 await が必要です。

この関数は interaction への応答を消費します。 呼んだ後に同じ interaction へ response.send_message() することはできないため、 権限チェックは応答を返す前(ハンドラの先頭)で行い、呼んだら return してください。

core.state.ctx(共有状態・読み取り用)

from core.state import ctx

bot 全体の共有状態です。拡張機能からは読み取り専用として扱ってください。 値を書き換えると bot 本体の動作に直接影響します。

属性型内容
ctx.extension_commands_groupapp_commands.Groupこの拡張機能用のコマンドグループ。ロード時のみセットされ、ロード完了後はクリアされるため、モジュール先頭で変数に保持してください。
ctx.extension_loggerlogging.Loggerこの拡張機能用のロガー(同上)。logger.getChild("名前") で子ロガーも作れます。
ctx.server_pathpathlib.Path管理対象サーバーが存在するフォルダのパス。
ctx.server_namestrサーバー本体のファイル名(.config の server_name)。
ctx.paths.basepathlib.Pathbot(server.py)が設置されているフォルダのパス。
ctx.text.langstr現在の表示言語("ja" / "en")。

v2 からの移行

v2 では from __main__ import extension_commands_group のように __main__ からインポートしていましたが、v3 では core.state の ctx 経由に変更されています。v2 向けの拡張機能はそのままでは動作しません。 

# 旧 (v2)
from __main__ import extension_commands_group as tree
from __main__ import extension_logger

# 新 (v3)
from core.state import ctx
tree   = ctx.extension_commands_group
logger = ctx.extension_logger
